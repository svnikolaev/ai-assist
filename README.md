# ai-assist

**ai-assist** — универсальный терминальный ассистент на базе больших языковых моделей (LLM) с поддержкой встроенных и внешних инструментов. Проект вдохновлён [ai-shell](https://github.com/svnikolaev/ai-shell), но расширяет его концепцию на произвольные запросы и интеграцию с локальными моделями (Ollama, LM Studio). Модель может вызывать инструменты, реализованные прямо в приложении или описанные в конфигурации, что позволяет автоматизировать широкий круг задач.

## Возможности

- Универсальный ассистент — отвечает на любые вопросы, генерирует код, объясняет концепции и т.д.
- Поддержка **инструментов (tools)** — модель может вызывать функции для получения дополнительной информации или выполнения действий.
- **Встроенные инструменты** на Rust (безопасные и быстрые):
  - Калькулятор (`calculator`)
  - Выполнение shell‑команд с проверкой по стоп‑листу (`run_shell`)
  - Текущие дата и время (`current_time`)
- **Внешние инструменты**, определяемые в конфигурации: вызов произвольных команд, API, скриптов.
- Поддержка **нескольких бэкендов** с резервированием (опрашиваются по очереди до первого успешного).
- Работа с любыми OpenAI‑совместимыми API, включая локальные серверы:
  - [Ollama](https://ollama.com/) (с эндпоинтом `/v1/chat/completions`)
  - [LM Studio](https://lmstudio.ai/)
  - Облачные провайдеры (RouterAI, GPTunneL, OpenRouter и др.)
- Передача произвольных параметров модели (например, `nothink` для Qwen, температура и т.д.) через поле `options` в конфиге бэкенда.
- Кэширование ответов (на основе хеша запроса) для ускорения повторных обращений.
- Режим объяснения (`-e`) — показывает детали выполнения инструментов.
- Стоп‑лист для блокировки опасных команд (используется в инструменте `run_shell`).
- Удобный CLI с поддержкой аргументов, флагов и чтения запроса из `stdin`.
- Один бинарник без зависимостей (Rust).

## Установка

### Из исходников

```bash
git clone https://github.com/yourusername/ai-assist.git
cd ai-assist
cargo build --release
sudo cp target/release/ai-assist /usr/local/bin/ai
```

### Через cargo (если репозиторий опубликован)

```bash
cargo install --git https://github.com/yourusername/ai-assist.git
```

После установки проверьте:

```bash
ai --help
```

## Конфигурация

При первом запуске создаётся конфигурационный файл `~/.config/ai-assist/config.toml` с настройками по умолчанию. Вы можете отредактировать его под свои нужды.

### Структура конфига

```toml
# Язык объяснений (пока не используется, задел на будущее)
explain_language = "ru"

# Стоп-лист для опасных команд
stop_list = ["rm -rf /", "mkfs"]

# Системный промпт по умолчанию (можно переопределить флагом -p)
default_prompt = "Ты — полезный ассистент. Отвечай кратко и по делу."

# Список бэкендов (опрашиваются по порядку)
[[backends]]
api_url = "http://localhost:11434/v1/chat/completions"   # Ollama
api_key = ""
model = "qwen2.5:0.5b"
timeout_secs = 30

[backends.options]          # Дополнительные параметры для этого бэкенда
nothink = true              # Отключает блок  у Qwen
temperature = 0.7

# Можно добавить второй бэкенд как запасной
[[backends]]
api_url = "http://localhost:1234/v1/chat/completions"   # LM Studio
api_key = ""
model = "local-model"
timeout_secs = 30
# У этого бэкенда нет options

# Внешние инструменты (опционально)
[[tools]]
name = "weather"
description = "Получить погоду для города"
schema = { type = "object", properties = { city = { type = "string" } }, required = ["city"] }

[tools.execution]
type = "api_call"
url = "https://wttr.in/{city}?format=%t"
method = "GET"
```

### Описание полей

- **`default_prompt`** — системный промпт, если не передан через флаг `-p`.
- **`explain_language`** — язык для пояснений (пока зарезервировано).
- **`stop_list`** — массив строк (опасные паттерны). Если команда содержит любой из них, инструмент `run_shell` её заблокирует.
- **`backends`** — массив бэкендов. Каждый бэкенд содержит:
  - `api_url` — полный URL эндпоинта (должен поддерживать OpenAI‑совместимый API).
  - `api_key` — ключ API (для локальных серверов можно оставить пустым).
  - `model` — идентификатор модели.
  - `timeout_secs` — таймаут запроса.
  - `options` — произвольные параметры, которые будут переданы в тело запроса в поле `options` (специфично для Ollama, но может использоваться и другими бэкендами).
- **`tools`** — массив внешних инструментов. Каждый инструмент содержит:
  - `name` — уникальное имя (не должно совпадать со встроенными).
  - `description` — описание для модели.
  - `schema` — JSON Schema, описывающая параметры инструмента.
  - `execution` — способ выполнения. Поддерживаются три типа:
    - `type = "command"` — выполнение shell‑команды с подстановкой `{param}`.
    - `type = "api_call"` — HTTP‑запрос (пока только GET).
    - `type = "script"` — запуск внешнего скрипта.

### Переменные окружения

- `AI_API_KEY` — если задана, подставляется в поле `api_key` для **первого** бэкенда (только если у него не указан свой ключ).

## Использование

```bash
# Простой запрос
ai "какая сейчас дата?"

# Запрос с объяснением (показывает детали выполнения инструментов)
ai -e "сколько будет 25 * 4 + 10?"

# Переопределить системный промпт
ai -p "Ты — эксперт по Python" "напиши функцию для чтения CSV"

# Чтение запроса из stdin
echo "выполни ls -la" | ai

# Игнорировать кэш
ai -c "придумай название для стартапа"

# Показать справку
ai --help
```

### Примеры запросов

```bash
# Модель может вызвать встроенный калькулятор
ai "вычисли 2^10"

# Запрос, требующий выполнения shell-команды
ai "сколько файлов в текущей папке?"

# Запрос к внешнему инструменту (погода)
ai "какая погода в Лондоне?"

# Сложный запрос с несколькими инструментами (если модель поддерживает)
ai "получи текущее время и скажи, сколько дней до нового года"
```

## Встроенные инструменты

Все встроенные инструменты реализованы на Rust, безопасны и всегда доступны. Они описываются в коде, но модель получает их описания автоматически.

| Имя            | Описание                                                                 | Параметры                          |
|----------------|--------------------------------------------------------------------------|------------------------------------|
| `calculator`   | Вычисляет арифметические выражения (через `meval`).                      | `expression` (string)              |
| `run_shell`    | Выполняет команду в оболочке (`sh -c`) с проверкой по стоп‑листу.        | `command` (string)                 |
| `current_time` | Возвращает текущие дату и время (локальная таймзона).                    | нет параметров                     |

## Как добавить свой инструмент

### Встроенный инструмент (на Rust)

1. В файле `src/tools.rs` напишите функцию с сигнатурой `fn(&Value, &Config) -> Result<ToolOutput>`.
2. Добавьте описание инструмента в массив `builtins_list` в `ToolRegistry::new()` (как показано в коде).
3. Перекомпилируйте проект.

### Внешний инструмент (через конфиг)

Добавьте секцию `[[tools]]` в `config.toml` с описанием и способом выполнения. Поддерживаются три типа:

#### Команда

```toml
[[tools]]
name = "disk_usage"
description = "Показать использование диска для указанного пути"
schema = { type = "object", properties = { path = { type = "string" } }, required = ["path"] }

[tools.execution]
type = "command"
command = "du -sh {path}"
```

#### API‑вызов

```toml
[[tools]]
name = "joke"
description = "Получить случайную шутку"
schema = { type = "object", properties = {} }

[tools.execution]
type = "api_call"
url = "https://v2.jokeapi.dev/joke/Any?format=txt"
method = "GET"
```

#### Скрипт

```toml
[[tools]]
name = "backup"
description = "Запустить резервное копирование"
schema = { type = "object", properties = { target = { type = "string" } }, required = ["target"] }

[tools.execution]
type = "script"
path = "/home/user/scripts/backup.sh"
interpreter = "bash"
```

## Диагностика

Для проверки конфигурации и доступности бэкендов можно использовать:

```bash
ai --test   # (если реализовано, в текущей версии флага нет, можно добавить)
```

Пока рекомендуется просто выполнить простой запрос и следить за выводом ошибок.

## Требования

- Для работы с локальными моделями: установленный и запущенный сервер Ollama или LM Studio с загруженной моделью.
- Для внешних инструментов, вызывающих API, может потребоваться доступ в интернет.
- Rust (только для сборки из исходников).
